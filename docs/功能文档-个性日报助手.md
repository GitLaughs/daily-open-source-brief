# 个性日报助手功能文档

更新时间：2026-05-25

## 1. 当前定位

本项目已从“开源仓库日报”扩展为“个人技术与校园日报助手”：

- 保留原有 GitHub 高星项目抓取、SQLite 归档、SMTP 邮件、飞书发送和 cc-switch LLM 轮换方式。
- 新增西安交通大学公开官网信息源抓取，不需要登录。
- 新增 RSS/Atom 采集，用于通用新闻、半导体、EDA、电子工程资讯。
- 新增校园通知、开源项目、来源状态、候选快照的结构化 HTML 输出。
- 新增官网源解析测试和渲染测试，方便后续继续加源。

## 2. 本次新增功能

### 2.0 插件管理器

配置位置：`config/plugins.yml`。

核心实现：

- `app/plugins/base.py`：`PluginContext`、`PluginResult`、`BasePlugin`。
- `app/plugins/manager.py`：插件配置加载、启停、按阶段执行。
- `app/plugins/registry.py`：插件元数据、注册表、配置校验、启停写回。
- `app/plugins/local_loader.py`：加载 `plugins/local/*.py` 本地插件。
- `app/plugins/builtins.py`：当前内置插件 adapter。
- `app/runner.py`：可复用执行入口，供旧 `run_daily` 和新 CLI 调用。
- `app/cli.py`：插件和配置管理命令。

当前阶段：

| 阶段 | 用途 | 当前插件 |
| --- | --- | --- |
| `provider` | 配置 LLM/provider | `ccswitch` |
| `collector` | 采集候选内容 | `github`、`webpage`、`rss` |
| `summarizer` | 生成日报正文 | `summarizer` |
| `renderer` | 写 HTML 归档和 digest | `renderer` |
| `sender` | 投递渠道 | `lark`、`mail` |

新增功能默认应加到 `app/plugins/`，并在 `config/plugins.yml` 提供开关和参数；`app/run_daily.py` 只保留编排逻辑。

当前 CLI：

```powershell
python -m app.cli run --sample --skip-web --skip-rss --skip-mail --skip-lark --force-send
python -m app.cli plugin list
python -m app.cli plugin enable rss
python -m app.cli plugin disable mail
python -m app.cli plugin check
python -m app.cli plugin status
python -m app.cli config validate
python -m app.cli config show
```

插件运行会写入 `plugin_runs` 和 `plugin_health`，用于后续 Web 控制台展示最近状态、失败次数、耗时和条目数。

### 2.1 西交官网公开信息源

配置位置：`config/sources.yml` 的 `webpages`。

当前启用：

| 名称 | 页面 | 用途 |
| --- | --- | --- |
| `xjtu-jwc-teaching-notice` | `https://jwc.xjtu.edu.cn/jxxx/jxtz2.htm` | 教务处教学通知 |
| `xjtu-graduate-notice` | `https://gs.xjtu.edu.cn/tzgg.htm` | 研究生院通知公告 |
| `xjtu-student-affairs-notice` | `https://xsc.xjtu.edu.cn/xgdt/tzgg.htm` | 学工部通知公告 |
| `xjtu-news-home` | `https://news.xjtu.edu.cn/` | 学校新闻网首页 |

实现文件：`app/fetch_webpage.py`。

特性：

- 使用 `requests` 抓取静态 HTML，不引入浏览器常驻进程。
- 支持西交部分站点的轻量 `dynamic_challenge` cookie 验证。
- 按 `<li>` / `<tr>` 中的链接、标题、日期提取条目。
- 按 URL 去重。
- 按日期、关键词和行动属性评分。
- 单个来源失败只进入“来源状态”，不阻断 GitHub、邮件或飞书发送。

### 2.2 今日优先处理

LLM 和模板摘要都会把校园官网、RSS 产业资讯里更需要处理或更有价值的条目放在更靠前的位置。

模板结构：

1. 今日优先处理
2. 校园与产业资讯雷达
3. 今日最值得看的开源项目
4. GitHub 高星项目观察
5. 可后续尝试的项目
6. 来源状态

评分偏好：

- 新近条目优先。
- `报名`、`申报`、`选课`、`考试`、`毕业`、`确认`、`办理`、`公示`、`竞赛`、`讲座`、`答辩`、`奖学金`、`助学` 等行动型通知优先。
- 每个来源可配置 `priority_keywords`，例如教务处偏向 `选课`、`考试`、`课程`。

### 2.3 RSS/Atom 产业资讯源

配置位置：`config/sources.yml` 的 `rss`。

当前启用：

| 名称 | 页面 | 用途 |
| --- | --- | --- |
| `semiconductor-engineering` | `https://semiengineering.com/feed/` | 半导体、EDA、验证、制造资讯 |
| `edn` | `https://www.edn.com/feed/` | 电子工程、模拟、电源、嵌入式资讯 |
| `ieee-spectrum` | `https://spectrum.ieee.org/rss/fulltext` | 通用技术、工程、计算趋势 |

实现文件：`app/fetch_rss.py`。

特性：

- 支持 RSS 2.0 和 Atom。
- 使用标准库 XML 解析，不新增依赖。
- 按标题、链接、发布时间、摘要生成统一条目。
- 按行业关键词、发布时间和来源权重评分。
- 单个 RSS 超时或失败只写来源状态，不阻断日报。

### 2.4 来源健康与新增判断基础

实现文件：`app/db.py`。

新增数据基础：

- `source_runs`：记录每次来源抓取的状态、条目数、耗时、错误。
- `source_health`：保存每个来源的累计成功/失败、最近状态、最近成功/失败时间。
- `items.first_seen_at`：条目首次发现时间。
- `items.last_seen_at`：条目最近抓到时间。
- `items.last_notified_slot`：后续用于高优先级补推和去重提醒。

用途：

- `send-only` 也能显示来源健康。
- 后续可以判断“今日新增”。
- 飞书分级推送不需要重新猜测哪些内容是新内容。

### 2.5 HTML 输出优化

实现文件：`app/render.py`。

本次修复：

- 原来 markdown 的 `-` 会直接变成裸 `<li>`，现在会正确包成 `<ul class="digest-list">`。
- 缩进说明会放进对应列表项内部。
- URL 自动变成可点击链接。
- 新增顶部指标：网页/RSS 条目数、开源候选项目数、最高 Stars、输出模式。
- 新增结构化区块：今日简报、网页与 RSS 情报源、开源项目速览、来源状态、候选仓库快照。

设计约束：

- 白色和浅灰背景。
- 无渐变、无玻璃态、无装饰性大背景。
- 卡片只用于单条通知和单条项目。
- 移动端自动变成单列。

### 2.6 新命令参数

`app.run_daily` 新增：

```bash
python -m app.run_daily --skip-web
python -m app.run_daily --skip-rss
```

用途：

- 跳过 `webpages` 官网源。
- 跳过 `rss` RSS/Atom 源。
- 适合离线 smoke、只测 GitHub、或临时排查学校站点网络问题。

可选归档清理：

```bash
python -m app.run_daily --archive-retention-days 180
```

也可以通过环境变量启用：

```text
DAILY_BRIEF_ARCHIVE_RETENTION_DAYS=180
```

默认值是 `0`，表示不自动删除归档。清理只会删除 `public/archive/YYYY-MM-DD.html` 这种日期命名文件，不会处理其它 HTML。

原有授权和发送方式未改：

- GitHub 仍用 `GITHUB_TOKEN`。
- SMTP 仍用 `SMTP_HOST` / `SMTP_USER` / `SMTP_PASS` 等变量。
- 飞书仍用 `LARK_SEND` / `LARK_AS` / `LARK_USER_ID` / `LARK_CHAT_ID`。
- cc-switch 仍用原有 `LLM_FROM_CCSWITCH` 逻辑。
- 飞书发送失败现在只记录日志，后续 SMTP 邮件和本地归档继续执行。

## 3. 怎么使用

### 本地完整 smoke

会使用样例 GitHub 仓库，同时抓取官网公开页面，不发邮件、不发飞书：

```powershell
python -m app.run_daily --sample --skip-mail --skip-lark --force-send
```

### 本地离线 smoke

不访问 GitHub API，也不访问官网源或 RSS：

```powershell
python -m app.run_daily --sample --skip-web --skip-rss --skip-mail --skip-lark --force-send
```

### 只采集，不发送

```powershell
python -m app.run_daily --collect-only
```

### 用已采集候选生成日报

```powershell
python -m app.run_daily --send-only
```

输出归档：

```text
public/archive/YYYY-MM-DD.html
```

## 4. 添加新的官网源

只改 `config/sources.yml`：

```yaml
webpages:
  - name: xjtu-example
    title: 西交某官网 通知公告
    url: https://example.xjtu.edu.cn/tzgg.htm
    source_type: school_notice
    category: academic
    enabled: true
    limit: 10
    weight: 5
    url_allow_patterns:
      - /info/
    priority_keywords:
      - 报名
      - 申报
      - 答辩
```

字段说明：

- `name`：数据库来源名，稳定后不要随意改。
- `title`：日报里显示的来源名。
- `url`：公开列表页。
- `source_type`：目前建议保持 `school_notice`。
- `category`：分类标签。
- `enabled`：临时关闭某源时改成 `false`。
- `limit`：每个来源最多取几条，建议 10 到 12。
- `weight`：来源权重，教务/培养类可略高。
- `url_allow_patterns`：只保留真实内容页，西交站点常用 `/info/`。
- `priority_keywords`：命中后加权。

全局校园关键词可放在 `config/sources.yml`：

```yaml
school:
  priority_keywords:
    - 选课
    - 考试
    - 毕业
```

全局关键词会和单个来源的 `priority_keywords` 合并。

## 5. 添加新的 RSS/Atom 源

只改 `config/sources.yml`：

```yaml
rss:
  - name: example-industry-feed
    title: Example Industry Feed
    url: https://example.com/feed/
    source_type: industry_news
    category: semiconductor
    enabled: true
    limit: 10
    weight: 5
    tags:
      - semiconductor
      - eda
    priority_keywords:
      - EDA
      - verification
```

全局产业关键词可放在：

```yaml
industry:
  priority_keywords:
    - semiconductor
    - chip
    - EDA
```

RSS 源建议先少量启用。外部站点慢或限流时，健康表会记录失败，不影响其它来源。

## 6. 服务器资源策略

保持轻量：

- 官网抓取只用 `requests`。
- 不常驻浏览器。
- 不做全文索引。
- 单源失败不重试过多。
- 每个来源默认只取十几条。
- SQLite 继续作为唯一存储。
- RSS/Atom 不下载正文附件，只保存标题、链接和摘要。

建议后续再加：

- 归档保留天数清理。
- RSS 源解析。
- 每周汇总模式。
- 来源健康日报。
- 对重要通知做截止日期抽取。
