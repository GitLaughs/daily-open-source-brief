# 2026-05-25 执行计划：个性日报助手扩展

## 目标

在不改变现有 GitHub、SMTP、飞书、LLM 授权方式的前提下，把项目扩展成更完整的个性日报助手：

1. 接入西安交通大学公开官网平台。
2. 优化当前 HTML 输出格式。
3. 增加低服务器负担的实用功能。
4. 持续更新功能文档，方便后续接手。

## 计划

### 第一步：确认边界

- 保留 `app.mailer`、`app.lark_sender`、`app.ccswitch` 的现有授权和发送逻辑。
- 新增官网源只通过 `config/sources.yml` 配置。
- 单个官网源失败不能影响日报生成和发送。

结果：已完成。

### 第二步：官网源接入

- 新增 `app/fetch_webpage.py`。
- 默认接入教务处、研究生院、学工部、新闻网。
- 支持西交站点公开页面的轻量 `dynamic_challenge`。
- 抽取标题、链接、日期、摘要、来源。
- 写入现有 `items` 表，不新增复杂服务。

结果：已完成。

### 第三步：日报内容结构升级

- `app/summarize.py` 同时接收 GitHub 仓库和校园官网条目。
- 模板摘要新增“今日优先处理”和“西交官网通知雷达”。
- LLM 输入新增 `campus_items` 和 `source_errors`。

结果：已完成。

### 第四步：输出格式优化

- 修复裸 `<li>` 输出。
- 增加顶部指标。
- 增加校园通知卡片、开源项目卡片、来源状态区块。
- 保持邮件 HTML 简洁，避免重型前端依赖。

结果：已完成。

### 第五步：低负担实用扩展

- 增加 `--skip-web`，用于离线 smoke 或临时跳过官网源。
- 官网源按关键词和日期评分，减少低价值信息。
- 来源错误可见，不需要翻日志才能知道哪个站点异常。
- 增加全局校园关键词 `school.priority_keywords`。
- 增加可选归档清理 `--archive-retention-days` / `DAILY_BRIEF_ARCHIVE_RETENTION_DAYS`。

结果：已完成。

### 第六步：验证和文档

- 新增官网解析测试。
- 新增 HTML 渲染测试。
- 更新 README。
- 新增 `docs/功能文档-个性日报助手.md`。

结果：已完成。

## 后续建议

优先级从高到低：

1. 截止日期抽取：从通知正文中识别报名/申报截止时间。
2. 每周汇总：把一周内未处理的重要通知和开源项目汇总一次。
3. RSS 源解析：接入少量高质量技术博客、半导体资讯。
4. 个人规则文件：把“必须提醒/忽略/降权”的关键词独立放入 `config/profile.yml`。
